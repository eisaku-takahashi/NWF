# NWF Spec Roadmap v2.0.1

---

## 1. 概要

本ドキュメントは NWF (Narrative Workflow Framework / Story OS) の将来計画書であり、  
Spec の進化、AI Workflow の成熟、Story Production Pipeline の運用、  
および Governance / Maintenance を統合した **NWF 全体の戦略ロードマップ** を定義する。

NWF v2.0.1 において Spec Foundation が完成したため、  
本ロードマップでは v2.x から v3.0.0 に至るまでのシステム進化フェーズと運用計画を定義する。

本ドキュメントは以下を目的とする。

- NWF システム進化フェーズの定義
- Spec ライフサイクルと Version Roadmap の整理
- AI Workflow 成熟度モデルの定義
- Story Production Pipeline のマイルストーン整理
- Governance / Maintenance の運用スケジュール定義
- v3.0 以降の将来展望の整理

---

## 2. Vision Statement

NWF の最終目標は以下である。

**「AI と人間が共生し、仕様・世界設定・物語・データが再帰的に整合性を保ちながら進化し続ける Story OS を構築すること」**

この Story OS は次の特性を持つ。

- Spec Driven Development
- AI Assisted Content Generation
- Human-in-the-Loop Governance
- Recursive Integrity Validation
- Version Controlled World Building
- Long-form Story Production Pipeline
- Knowledge / Timeline / World Data の統合管理
- Git による完全履歴管理
- Story と Spec の相互進化

最終形は **Fully Integrated Story OS** と呼ばれる。

## 4. Spec & AI Maturity Roadmap

### 4.1 AI Workflow Maturity Model

AI 活用成熟度を以下の 6 段階で定義する。

Level 1: Prompt Driven
- 単発プロンプトによる生成
- 人間が全体設計

Level 2: Template Driven
- プロンプトテンプレート使用
- Spec 作成支援

Level 3: Workflow Driven
- 定義された Workflow に従って AI が生成
- Draft → Review → Rewrite

Level 4: Engine Assisted
- Data / Engine / Validation を AI が利用
- 整合性チェック付き生成

Level 5: Semi-Automated Story Generation
- Story Draft を半自動生成
- Timeline / Character / Event 自動整合

Level 6: Recursive Story OS
- Spec / Data / Story / Workflow が再帰的に改善
- AI が Integrity を維持
- Human は Governance と最終意思決定

NWF の目標:
**Level 6 到達**
