# NWF Emotional Curve Model v1.0

## 概要

NWF（Novel Writing Framework）における Emotional Curve Model は、物語における感情の変化を構造的に表現・管理するためのモデルである。

物語は出来事の連続だけではなく、読者およびキャラクターの感情の変化によって強く印象づけられる。Emotional Curve Model はその感情の起伏を数値化および構造化することで、物語の感情的な流れを分析可能な形にすることを目的とする。

このモデルにより、以下を実現する。

物語全体の感情構造の可視化  
キャラクター感情の変化の追跡  
シーンごとの感情強度の管理  
物語設計の補助  


## Emotional Curve Model の役割

Emotional Curve Model は NWF 内で以下の役割を持つ。

物語の感情的な起伏の記録  
シーン単位の感情強度の定義  
キャラクター感情の変化の追跡  
読者体験の設計支援  


## Emotional Curve の基本概念

Emotional Curve は物語の進行に伴う感情の強弱を曲線として表現する概念である。

一般的に物語では以下のような感情の流れが発生する。

導入による感情の形成  
葛藤による感情の上昇  
危機による感情のピーク  
解決による感情の収束  

Emotional Curve Model はこれらの変化をデータとして管理する。


## 感情強度

感情強度は物語のある時点における感情の強さを示す指標である。

感情強度は数値として表現される。

低い数値は穏やかな感情状態を示す。  
高い数値は強い緊張や感情的衝突を示す。


## 感情の種類

物語における感情は複数の種類に分類できる。

主な感情の例

喜び  
悲しみ  
怒り  
恐怖  
驚き  
期待  

Emotional Curve Model では必要に応じてこれらを分類して管理する。


## Scene と Emotional Curve

Scene は Emotional Curve の主要な構成単位である。

各 Scene には以下の情報が関連付けられる。

感情強度  
感情の種類  
感情変化の方向  


## Beat と感情変化

Beat は Scene 内の最小単位の出来事であり、感情変化の直接的な原因となる。

Beat によって以下の変化が発生する。

キャラクターの感情変化  
緊張の増加または減少  
物語の感情的転換  


## Character と Emotional Curve

キャラクターは Emotional Curve の重要な要素である。

各キャラクターは物語の進行に伴い感情状態が変化する。

Emotional Curve Model は以下を管理できる。

キャラクターの感情状態  
感情変化のタイミング  
感情の蓄積と解放  


## Thread と Emotional Curve

Thread は物語のストーリーラインであり、それぞれ独自の Emotional Curve を持つことができる。

複数の Thread が存在する場合、それぞれの感情曲線が相互に影響することで物語全体の感情構造が形成される。


## Emotional Curve の可視化

Emotional Curve はグラフ形式で可視化できる。

横軸は物語の進行  
縦軸は感情強度  

この可視化により以下が可能となる。

物語の盛り上がりの確認  
感情ピークの配置分析  
物語テンポの調整  


## AI連携

Emotional Curve Model は AI による物語分析および生成に利用できる。

AIは以下の用途でこのモデルを利用する。

物語の感情構造分析  
感情的クライマックスの検出  
感情バランスの調整提案  
ストーリー生成の支援  


## Emotional Curve Model の設計思想

このモデルは以下の設計思想に基づく。

物語感情の構造化  
読者体験の分析可能化  
キャラクター感情の追跡  
AIによる物語分析の支援  


## まとめ

Emotional Curve Model は NWF において物語の感情構造を管理するための重要なモデルである。

このモデルにより物語の感情的な流れを可視化・分析でき、より効果的なストーリー設計を支援する。
