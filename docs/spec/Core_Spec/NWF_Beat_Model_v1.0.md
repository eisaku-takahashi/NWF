# NWF Beat Model v1.0

## 概要

NWF（Novel Writing Framework）において **Beat** は物語の最小ドラマ単位である。

BeatはSceneの内部で発生する小さな変化を表し、キャラクターの行動、
感情の変化、情報の提示などを通じて物語のテンポと感情の流れを形成する。

本ドキュメントでは NWF における Beat の構造と役割を定義する。

---

# Beat の基本概念

BeatはSceneを構成する最小のドラマ要素である。

Sceneが「出来事の単位」であるのに対して、
Beatは「変化の単位」である。

例

- 新しい情報が提示される
- キャラクターの感情が変化する
- 行動が開始される
- 対立が発生する
- 決断が行われる

これらの小さな変化が連続することでSceneが形成される。

---

# Beat と Scene の関係

BeatはSceneの内部構造を構成する。

構造

Scene
↓
Beat 1
Beat 2
Beat 3
Beat 4

複数のBeatが連続することで一つのSceneが完成する。

---

# Beat Structure

Beatは通常以下の情報を持つ。

- Beat ID
- Type
- Description
- Related Characters
- Emotional Change
- Result

これらの情報によってBeatの内容が定義される。

---

# Beat の役割

Beatは物語の細かな変化を表す。

主な役割

- 情報提示
- 感情変化
- 行動開始
- 対立発生
- 意思決定

Beatの連続によって物語のリズムが生まれる。

---

# Beat のタイプ

Beatは役割によって分類することができる。

例

Information Beat  
新しい情報が提示される

Emotional Beat  
キャラクターの感情が変化する

Action Beat  
キャラクターが行動する

Conflict Beat  
対立が発生する

Decision Beat  
キャラクターが決断を下す

---

# Beat と Character

Beatはキャラクターの行動や感情変化を通じて発生する。

Character  
↓  
Action / Emotion  
↓  
Beat

このようにBeatはキャラクターの内面や行動と密接に関係する。

---

# Beat と物語テンポ

Beatの配置は物語のテンポに大きく影響する。

例

短いBeatが連続する場合  
→ テンポが速くなる

長い説明的Beatが続く場合  
→ テンポが遅くなる

Beat構造を調整することで物語のリズムを制御できる。

---

# Beat Model の目的

NWF Beat Modelの目的は次の通りである。

- Scene内部のドラマ構造を明確化する
- キャラクター行動と感情変化を管理する
- 物語テンポを分析する
- AIによる物語生成を支援する

Beat Modelは NWF Scene Model を補完する構造として機能する。
