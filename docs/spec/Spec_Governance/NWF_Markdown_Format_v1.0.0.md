# NWF Markdown Format v1.0.0

---

## 1. 概要

本ドキュメントは  
NWF Specおよび関連ドキュメントにおける  
**Markdown記述フォーマットの標準構造**を定義する。

目的

- Spec構造の統一
- 可読性の向上
- AI生成ドキュメントの品質安定

---

## 2. 基本テンプレート

NWF Markdownドキュメントは  
以下のテンプレートを使用する。

Source: docs/spec/<Spec_Category>/<File_Name>.md  
Updated: YYYY-MM-DDTHH:MM:SS+09:00  
PIC: Engineer / ChatGPT  

# Document Title vX.X.X

ドキュメントの目的を説明する。

---

## 2. Section

仕様または説明を記述する。

---

## 3. まとめ

ドキュメントの要点をまとめる。

---

---

## 3. 見出しルール

見出しは以下の階層で使用する。

#  
ドキュメントタイトル

##  
セクション

###  
サブセクション

---

## 4. セクション構造

Specドキュメントは  
基本的に以下の流れを採用する。

1 概要  
2 仕様または説明  
3 まとめ

必要に応じてセクションを追加できる。

---

## 5. バージョン表記

ドキュメントタイトルには  
必ずバージョン番号を含める。

例

NWF_File_System v2.0.0  
ThreadEngine Spec v1.2.0

---

## 6. まとめ

本フォーマットは  
NWFドキュメントの統一的構造を定義するものであり

- Spec
- Engine
- Plugin
- Architecture

すべてのMarkdownドキュメントに適用される。
